# INTERMEDIATE Day 09 — Design Patterns II: Observer, Strategy, Decorator

## Objectives
- Implement three fundamental behavioral/structural patterns
- Know when each pattern applies

## Patterns

| Pattern | Problem It Solves | Real-World Example |
|---------|------------------|--------------------|
| **Observer** | Notify multiple objects when state changes | Event listeners, pub/sub, MVC updates |
| **Strategy** | Swap algorithms at runtime | Sorting strategies, payment methods, auth policies |
| **Decorator** | Add behavior to objects without modifying them | Java I/O streams, logging wrappers, middleware |

```
javac *.java
java Lesson
```
