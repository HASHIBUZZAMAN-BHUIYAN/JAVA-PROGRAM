# INTERMEDIATE Day 08 — Design Patterns I: Singleton, Factory, Builder

## Objectives
- Implement and understand three foundational creational patterns
- Know when (and when NOT) to use each

## Patterns

| Pattern | Problem It Solves | When to Use |
|---------|------------------|-------------|
| **Singleton** | Ensure only one instance exists | Logging, config, DB connection pool |
| **Factory** | Decouple object creation from usage | When the type to create depends on runtime data |
| **Builder** | Construct complex objects step by step | Objects with many optional fields |
